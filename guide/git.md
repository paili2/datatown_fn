### 1. 커밋 메시지 구조

```bash
[타입] : 변경 요약
```

- 예시:
  - `feat: 로그인 기능 추가`
  - `fix: 회원가입 오류 수정`
  - `docs: README 업데이트`

---

### 2. 커밋 타입 종류

| 타입     | 설명                     |
| -------- | ------------------------ |
| feat     | 새로운 기능 추가         |
| fix      | 버그 수정                |
| docs     | 문서 관련 변경           |
| style    | 코드 스타일, 포맷 등     |
| refactor | 리팩토링(동작 변화 없음) |
| test     | 테스트 코드 추가/수정    |
| chore    | 설정, 빌드, 기타         |
